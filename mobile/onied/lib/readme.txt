АРХИТЕКТУРА: MVVM
СТРУКТУРА ПРОЕКТА:
- app/                       -- входная точка, конфигурация всего приложения
- views/                     -- для виджетов, представляющих страницы (обычно начинаются со Scaffold)
- views/view/                -- для какой-то страницы
- views/view/view.dart       -- входная точка страницы (не содержит никакой бизнес-логики, берет инфу из view_model)
- views/view/view_model.dart -- модель страницы (берет инфу из репозиториев в виде моделей /models/)
- views/view/components/     -- для виджетов страницы (вспомогательные элементы, использующиеся только на этой странице)
- components/                -- для общих виджетов
- models/                    -- для моделей
- models/enums/              -- для перечислений моделей
- models/model/              -- для конкретной модели
- repositories/              -- для репозиториев (обрабатывают инфу из сервисов и представляют в виде моделей)
- services/                  -- для сервисов (получают инфу из API, не знают о моделях)
